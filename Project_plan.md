# План разработки Backend для IELTS Preparation App (NestJS)

## 1. Архитектура базы данных (PostgreSQL)

Для реализации истории и аналитики нам понадобится надежная реляционная структура.

### Основные сущности:
- **User**: Хранение данных пользователя (id, email, password, name).
- **Exam**: Заголовок модуля (например, "IELTS Academic Test 1").
- **Question**: Вопросы для Reading и Listening (текст, тип, правильный ответ, баллы).
- **Attempt**: Запись о прохождении теста. Связывает пользователя и экзамен.
  - `status`: IN_PROGRESS, COMPLETED.
  - `scores`: JSON или отдельные колонки (listening_score, reading_score, writing_score).
- **UserAnswer**: Конкретные ответы пользователя в рамках одной попытки (для работы над ошибками).
- **WritingSubmission**: Текст эссе, фидбек от AI и детальные оценки по критериям.

---

## 2. Структура модулей NestJS

Проект следует разбить на независимые модули для удобства поддержки:

1.  **AuthModule**: Регистрация, авторизация (JWT + Passport.js).
2.  **UsersModule**: Управление профилем пользователя.
3.  **ExamsModule**: CRUD для тестов (загрузка текстов для чтения, аудио для листенинга и вопросов).
4.  **AttemptsModule**: Главная бизнес-логика:
    - Создание попытки.
    - Прием ответов.
    - Завершение теста.
5.  **ScoringModule**: Логика проверки:
    - `Reading/Listening Service`: Сравнение строк/выбор ответов.
    - `Writing Service`: Интеграция с OpenRouter API.
6.  **AnalyticsModule**: Формирование данных для графиков (агрегация баллов по датам).
7.  **ExternalApiModule**: Обертка над HTTP-клиентом для запросов к OpenRouter.

## 3. Логика работы модулей

### 📥 Listening & Reading
1. Пользователь получает список вопросов через `ExamsModule`.
2. После завершения массив ответов отправляется в `ScoringModule`.
3. Бэкенд сравнивает ответы с `correctAnswer` в БД (учитывая регистр и лишние пробелы).
4. На основе количества правильных ответов (Raw Score) вычисляется Band Score (1.0 - 9.0).

### ✍️ Writing (AI Integration)
1. Пользователь отправляет текст эссе.
2. `WritingService` отправляет запрос в **OpenRouter** (модели типа GPT-3.5/4 или Claude).
3. **Prompt (Пример):**
   > "Оцени это эссе по стандартам IELTS. Верни ответ СТРОГО в формате JSON с полями: bandScore, taskResponseScore, coherenceScore, lexicalScore, grammarScore, feedback."
4. Полученный результат сохраняется в БД и привязывается к попытке.

### 📊 Аналитика и Графики
Для графиков фронтенду нужны данные в формате временного ряда.
- **Endpoint:** `GET /analytics/progress`
- **Логика:**
  - Группировка попыток пользователя за последние 30 дней.
  - Расчет среднего балла по каждому модулю за каждый день или по каждой попытке.
  - Возврат массива: `[{ date: '2023-10-01', listening: 6.5, reading: 7.0, writing: 5.5 }]`.

## 4. Этапы реализации

### Этап 1: Setup
- [ ] Инициализация NestJS.
- [ ] Настройка Prisma или TypeORM.
- [ ] Docker-compose для PostgreSQL.

### Этап 2: Пользователи и Доступ
- [ ] Авторизация (Sign up / Login).
- [ ] Защита роутов через `AuthGuard`.

### Этап 3: Контент и Прохождение
- [ ] Создание миграций для таблиц Exams и Questions.
- [ ] API для получения контента теста.
- [ ] Логика сохранения ответов.

### Этап 4: Проверка и AI
- [ ] Алгоритм для Reading/Listening (Band Score Converter).
- [ ] Интеграция OpenRouter для Writing.
- [ ] Обработка ошибок API (Retry логика).

### Этап 5: История и Статистика
- [ ] API для получения истории всех `Attempts`.
- [ ] Агрегационный запрос для графиков прогресса.

## 5. Стек технологий
- **Framework:** NestJS
- **DB:** PostgreSQL
- **ORM:** Prisma (рекомендуется для быстрой разработки)
- **AI API:** OpenRouter (модели: `gpt-3.5-turbo` или `google/palm-2`)
- **Documentation:** Swagger (@nestjs/swagger)
